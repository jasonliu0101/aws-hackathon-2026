PressPlay Academy Demo 前端規劃筆記
====================================

整理日期：2026-07-13
用途：建立 PressPlay Academy 類型的線上學習平台 Demo，先復刻產品骨架，再替換課程內容與功能。

一、產品定位
------------

PressPlay Academy 類型的平台可以視為以下四個產品組合：

1. 課程探索與內容商城
2. 我的學習與課程管理
3. 多媒體課程播放器
4. 學員、老師與社群互動

公開資料顯示，平台支援圖文、影片、音訊與 Podcast 等不同內容形式，並包含單次購買與訂閱型課程。使用者可以從「我的學習」進入已取得的課程，並依照內容類型、時間或排序方式篩選內容。

二、Demo 第一版核心流程
----------------------

首頁
  -> 瀏覽課程
  -> 查看課程詳情
  -> 加入我的學習
  -> 開啟課程內容
  -> 閱讀文章或播放影片／音訊
  -> 記錄學習進度

第一版建議先不做真正付款與完整後端，先確保以上流程可以完整操作。

三、建議頁面
------------

必要頁面：

- Splash / Login：啟動畫面與登入入口
- Home：推薦課程、熱門內容、分類入口
- Explore：課程探索與搜尋
- Course List：課程列表、分類篩選、排序
- Course Detail：課程介紹、老師資訊、章節預覽、加入學習
- My Learning：已加入或已購買的課程、學習進度
- Learning Room：課程章節與內容清單
- Lesson Detail：文章、影片或音訊內容頁
- Profile：會員資料與設定

第二階段功能：

- 真實購買流程
- 訂閱方案
- 收藏／追蹤課程
- 閱讀清單
- 課程筆記
- 講義或附件下載
- 作業繳交
- 課程留言
- 老師社團
- 私訊
- 通知中心

六、頁面路由
------------

/                         首頁
/login                    登入
/explore                  探索課程
/courses                  課程列表
/courses/:id              課程詳情
/my-learning              我的學習
/learning/:courseId       課程學習頁
/learning/:courseId/:id   單篇課程內容
/profile                  個人設定

七、資料流
----------

Page
  -> Controller / ViewModel
  -> Use Case
  -> Repository
  -> Mock Data、Local Storage 或 API

頁面不直接處理 API，也不直接操作本地資料庫。未來可以將 MockCourseRepository 替換成 ApiCourseRepository，而不需要大幅修改 UI。

八、Demo 初期資料策略
--------------------

先使用本地假資料：

mock/
|-- courses.json
|-- lessons.json
|-- users.json
`-- progress.json

第一版需要能操作的互動：

- 點擊課程卡片
- 查看課程詳情
- 加入我的學習
- 開啟課程章節
- 播放影片或音訊
- 閱讀文章
- 切換上一則／下一則內容
- 更新學習進度
- 收藏課程
- 搜尋課程
- 篩選課程分類

九、建議 Flutter 套件
--------------------

- go_router：路由管理
- Riverpod：狀態管理
- Dio：未來串接 API
- video_player：影片課程
- just_audio：音訊與 Podcast
- Hive 或 Isar：本地收藏與學習進度
- json_serializable：JSON 與資料模型轉換

十、開發順序
------------

1. 建立 App Shell、Theme 與路由。
2. 建立首頁與課程卡片。
3. 建立課程列表、搜尋、分類篩選。
4. 建立課程詳情頁。
5. 建立加入我的學習流程。
6. 建立課程章節與單篇內容頁。
7. 加入文章、影片、音訊三種內容。
8. 加入學習進度與收藏。
9. 再加入登入、付款、社群與通知。

十一、Demo 與正式產品的差異
--------------------------

Demo 可以使用：

- 假登入
- 本地 JSON
- 假購買按鈕
- 固定課程資料
- 本地影片或音訊
- 本地學習進度

正式產品才需要加入：

- 真實會員驗證
- 權限與付費內容控制
- 金流
- 後端 API
- CDN 與媒體串流
- 雲端學習進度同步
- 推播與通知
- 內容管理後台
- 社群與客服系統
