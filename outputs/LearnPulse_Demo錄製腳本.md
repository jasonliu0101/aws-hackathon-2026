# LearnPulse Demo 錄製腳本

## 核心故事

用一位學員的完整旅程講清楚：「平台不猜情緒，只觀察他在哪裡反覆倒帶；在正確時機接住他，並把集體卡點回饋給創作者。」三分鐘內先讓觀眾看見問題，再揭露技術與商業價值。

## 錄製前檢查

- 執行 `bash prototype/start.sh`。
- 確認 `http://localhost:8899/front/`、`http://localhost:8899/mid/`、`http://localhost:3000/` 都可開啟。
- 3000 port 不可有舊服務；目前跨頁連結硬編碼為 `localhost:3000`。
- 在 mid 開啟 `http://localhost:8899/mid/?selftest=1`，確認 13 項自測全為 PASS。
- 建議錄製 1280×720、瀏覽器縮放 100% 或 125%、游標放大、關閉系統通知與密碼提示。
- 先預載所有頁面與影片，關閉 DevTools；若狀態混亂，清除該站 `localStorage` 後重開。
- 建議用 OBS；麥克風峰值約 -12 dB，畫面轉場以硬切為主，字幕全程開啟。

## 3 分鐘主版本

| 時間 | 畫面與操作 | 旁白重點 |
|---|---|---|
| 0:00–0:20 | 先顯示簡報問題頁或 mid 影片時間軸；快速拖回同一熱點兩次 | 「線上學習平台知道你按了播放，卻不知道你為什麼一直倒帶。重播不是失敗，而是一個被忽略的求救訊號。」 |
| 0:20–0:45 | 開啟 `http://localhost:8899/front/?demo=result`，顯示測驗結果與推薦，點入影片 | 「課前先用偏好做輕量分流，但不貼標籤；推薦只決定起點。」 |
| 0:45–1:05 | 開啟 `http://localhost:8899/mid/?open=coach`，聚焦熱力圖與 AI 教練卡 | 「真正的核心在課中。我們找出多人反覆重播的秒數，等待七秒，確認學員沒有自行恢復後才介入。」 |
| 1:05–1:20 | 展開教練卡、顯示解釋與來源，再操作投票或測驗 | 「AI 只用當前字幕、章節與教材回答，並保留來源。學員可以關閉、略過或延後。」 |
| 1:20–1:35 | 顯示重播後恢復、跳至筆記或 `?open=notes` | 「我們不只量點擊，而是量介入後六十秒是否繼續看、是否完成章節、重播是否下降。」 |
| 1:35–1:50 | 開啟 `http://localhost:8899/mid/?open=reels&i=0`，展示精華片段 | 「同一個卡點也能被切成精華，讓學員用更低摩擦的方式重建理解。」 |
| 1:50–2:05 | 開啟 `http://localhost:8899/mid/creator.html`，滑過熱點與改課建議 | 「集體卡點回到創作者後台：先改影響人數最多、誤解最深、成本最低的內容。」 |
| 2:05–2:20 | 開啟 `http://localhost:3000/`，展示學習城市成長、任務或複習 | 「課後的完成與複習會讓學習城市成長，把一次介入變成持續前進的回饋。」 |
| 2:20–2:35 | 回到 mid 的排程頁 `?open=sched`，顯示召回內容與時間 | 「若學員離開，系統不是泛用提醒，而是帶他回到最後卡點的三十秒精華與下一步任務。」 |
| 2:35–2:50 | 切到 AWS 架構圖 | 「正式版會把 replay、介入與 recovery 事件送入 EventBridge；S3 與 Athena 做分析，Bedrock Knowledge Bases 提供有來源的 RAG，Scheduler 負責延遲召回。」 |
| 2:50–3:00 | 顯示結尾標語 | 「LearnPulse 先把 replay 埋點做真，再讓每一次卡住都能被接住。下一步是用一門課、四週 pilot 驗證 recovery。」 |

## 建議快速參數

- 課前結果：`/front/?demo=result`
- AI 教練卡：`/mid/?open=coach`
- 筆記：`/mid/?open=notes`
- Reels：`/mid/?open=reels&i=0`
- 召回排程：`/mid/?open=sched`
- 自動測試：`/mid/?selftest=1`

以上 mid 路徑前綴皆為 `http://localhost:8899`。

## 必須誠實說明的 Mock

建議在片尾或說明欄放一行：

> Prototype：486 人重播分布與投票為固定種子模擬；影片、字幕、章節、片段邊界與互動流程為可操作原型；AI 為預寫內容與本地檢索，尚未正式串接 Bedrock。

這不會削弱提案，反而能把評審注意力帶到真正的驗證計畫：先收真實 replay，再做介入 A/B。

## 90 秒備用剪輯

- 0:00–0:15：重播洞察與一句話問題。
- 0:15–0:45：熱力圖 → 七秒延遲 → AI 教練卡 → 測驗。
- 0:45–1:05：創作者後台與改課優先順序。
- 1:05–1:20：學習城市與精準召回。
- 1:20–1:30：AWS 架構與四週 pilot ask。

## 失敗備援

- 每個核心畫面事先錄 5–8 秒無聲 B-roll，現場網頁出錯時直接覆蓋。
- 截圖備份：AI 教練卡、創作者後台、AWS 架構、學習城市。
- 若影片 Range 載入失敗，避免即場除錯；直接切到已開啟的 `?open=coach` 分頁。
- 若 Next.js 被改到 3001，不要修改導頁流程；先停止占用 3000 的舊服務再重啟。
- 錄影最後保留 10 秒乾淨結尾，方便補字幕、Logo 與 CTA。
