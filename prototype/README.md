# PPA 播放介面原型 — 暫停熱力圖 × AI 助教 × 精華 Reels

> 一句話：**把「全班在哪裡按暫停」當成訊號，讓 AI 助教知道自己該在哪一秒出現。**

現行的 AI 助教活在側邊欄，等你去問它。但學習者卡住的時候不會發問 —— 他會**按下暫停**。
那一下暫停，就是他舉手的方式。這個原型把那隻手接起來。

---

## 怎麼跑

```bash
node prototype/serve.js      # → http://localhost:8899
```

（不要用 `python3 -m http.server`：它不支援 HTTP Range 請求，影片就沒辦法 seek，Reels 會直接廢掉。）

重新產生資料（改了熱點、章節、精華段落之後）：

```bash
python3 prototype/tools/build_data.py
```

## Demo 動線（照這個順序演，5 分鐘）

| # | 做什麼 | 台詞 |
|---|---|---|
| 1 | 指進度條上方那排柱子 | 「這是 486 位學員的暫停行為。峰值不是隨機的 —— 它落在**操作步驟**、**專有名詞**、**數字密集**的地方。」 |
| 2 | 播放，讓它跑到 **04:57** | AI 助教自己跳出來：「339 人在這一秒按了暫停（70%）」 |
| 3 | 按「為我解釋這段」 | 它講的不是摘要，是「**你為什麼卡在這**」—— 4 個動作參數要同時記，工作記憶裝不下 |
| 4 | **自己按一下暫停** | 熱力圖那一格當場長高、閃一下 → 「訊號是用出來的，不是猜出來的」 |
| 5 | 按「AI 助教整理筆記」 | 重點 / 名詞卡 / 數字牆 / **你卡住的地方** / 1 題知識挑戰 |
| 6 | 按「看精華 Reels」 | AI 把卡關點剪成 14–30 秒直式短片，滑著看。每則都能一鍵跳回完整課程 |
| 7 | 答完題 → 點右上角 🔔 | 推播排程：**今晚 21:30 / D+3 / D+7** |
| 8 | 按「模擬今晚 21:30 的推播」 | 收尾 |

**第 7 步是整個 demo 的重點。** 21:30 這個時間不是亂排的 ——
影片自己講了「晚上睡前那一次最重要」，所以推播就排在睡前。
那是使用者**真的會走去刷牙**的時刻。內容感知的排程，比「隔三天推一次」的罐頭高一個量級。

### Demo 捷徑

上台不用臨場點一輪，直接開對應畫面：

```
?open=coach     播到最高峰的卡關點，助教卡攤開
?open=notes     AI 筆記
?open=reels     精華 Reels（?i=2 指定第幾則）
?open=sched     推播排程
?open=push      推播卡
?t=297          跳到第幾秒
```

## 五個卡關點是怎麼來的

不是亂灑的。假設是：**學員按暫停，代表認知負荷超載，或想把這段記下來 / 跟著做。**
所以峰值照四種認知負荷長出來：

| | 時間 | 卡關類型 | 暫停率 | 內容 |
|---|---|---|---|---|
| h5 | 04:57 | **步驟要照做** | **70%** | 貝氏刷牙法：45 度、兩顆兩顆、來回 10 下、2–3 分鐘 |
| h6 | 05:40 | 步驟要照做 | 54% | 牙線要彎成 C 字形，上下刮 |
| h2 | 02:43 | 名詞沒定義 | 45% | 「牙齦溝」第一次出現，沒定義就往下講 |
| h4 | 03:58 | 來源想確認 | 42% | 引用 Cochrane 文獻回顧，「多減少 21%」是跟誰比？ |
| h1 | 01:24 | 數字記不住 | 34% | 80.4%、9 成、高出一倍 —— 三個數字，三種單位 |

**操作步驟的峰值最高**，因為那是唯一會讓人「暫停、放下手機、真的照著做一次」的內容。

### 這個假設站得住腳的證據

把這五個時間點的畫面截出來看 —— 它們**全部命中影片的圖卡**：

- `02:43` 畫面正在打「**牙齦溝**」字卡
- `03:58` 畫面是 **Cochrane Library 論文截圖 +「牙菌斑減少 21%」**
- `04:57` 是**貝氏刷牙法圖卡 +「牙刷跟牙齦呈 45 度」**
- `05:40` 是**牙線篇圖卡 +「C 字形」**

不是巧合：製作方也知道那幾段最難，才在那裡放圖卡。
**認知負荷是可以從逐字稿預測的** —— 這正是這套方法可以自動化的前提。

### 反向證據：業配段

`06:37–08:00` 的業配段在熱力圖上是一道**低谷**（斜線紋那段）。
沒人在那裡暫停 —— 但跳過率 87%。

這條低谷很重要：**如果暫停數只是「播放量的影子」，業配段不會塌下去。**
它塌了，代表這個訊號真的在量「認知負荷」，不是在量「有多少人在看」。

## 誠實揭露：哪些是真的，哪些是模擬的

| | 狀態 |
|---|---|
| 影片、逐字稿（320 條字幕）、時間軸 | **真的**（志祺七七《科學潔牙》，`sample/`） |
| 章節切分 | **真的** —— 影片自己有 8 張章節卡（逐字稿裡的 `【...】`） |
| 精華 clip 的起訖點 | **真的** —— 對齊到字幕邊界，每段都從完整句子起、完整句子收 |
| 486 位學員的暫停直方圖 | **模擬**（`build_data.py`，固定種子，可重現） |
| AI 助教的解說、筆記、考題 | **預先寫好的**，不是即時呼叫 LLM |

模擬的部分要換成真的，缺的不是模型，是**埋點**：
現在的行為表只有「進入 / 首次使用 / 關閉 / 返回」四種 AI 事件，
**沒有 `video_pause` 事件**，也沒有秒數。這套東西要落地，第一件事是補這個埋點。

## 檔案

```
prototype/
  index.html          播放頁
  styles.css
  app.js              播放器 / 熱力圖 / 助教 / 筆記 / Reels / 推播
  serve.js            支援 Range 的靜態伺服器
  data/course.json    ← build_data.py 產生
  assets/
    video.mp4         （硬連結到 sample/video.mp4）
    cover.jpg         影片封面
    thumbs/           7 個 clip 的代表幀（Reels 的模糊背景用）
  tools/build_data.py SRT → JSON、章節、熱力圖、精華段落、筆記、考題
```

素材：志祺七七《科學潔牙》，僅作 hackathon 原型展示。
